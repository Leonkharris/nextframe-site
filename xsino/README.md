# XSINO — Interactive Codex

Offline, single-file web UI for the XSINO brand codex. Mirrors the architecture of
Project Kinmuku's Interactive Codex (`build_*.py` → `data.js` → hash-router `index.html`),
bilingual **Spanish + English** instead of Chinese + English.

## How to open

- Double-click `index.html` — it runs fully offline (no build step, no server required).
- Or serve it locally: `python -m http.server 8124` from this folder, then open
  `http://localhost:8124/`.

## How to regenerate the data

`index.html` reads everything from `data.js` (`window.XSINO_DATA`), which is generated by
`build_xsino.py` — a pure-stdlib, zero-LLM, deterministic parser. Re-run it any time the
source files change:

```
python build_xsino.py
```

It parses:
- `99_Source_Files/bible/{brand_bible,pillars,visual_language,production_pipeline}.md`
  → `data.bible.{brand,pillars,visual,pipeline}`
- every `99_Source_Files/profiles/NN.md` → `data.characters`
- every `99_Source_Files/stories/NN.md` → `data.stories`

and prints a summary line (character/story/section counts) plus the size of the
written `data.js`. The curation layer (brand META, the 4 PILLARS, and the per-character
CURATION table — pillar, priority, colors, ES/EN alias) lives at the top of
`build_xsino.py`, the same pattern as Kinmuku's `ARCH`/`STATUS` dicts.

Source files marked with `【PROPUESTA】` / `[PROPOSAL]` are parsed with `proposal:true`
and rendered with a small PROPOSAL badge in the UI (the marker itself is stripped).

## Portrait / backdrop drop-in convention

Images are optional. If a file doesn't exist, the UI silently falls back to a generated
placeholder (via the image's `onerror` handler) — nothing needs to be reconfigured.

- **Character portraits:** drop a PNG at `assets/portraits/NN.png` (e.g. `assets/portraits/01.png`
  for Mateo Rivas). Falls back to a colored medallion with the character's initials
  (colors come from the CURATION table in `build_xsino.py`).
- **Pillar backdrops:** drop a PNG at `assets/pillars/{key}.png` where `{key}` is one of
  `inspire`, `love`, `anime`, `whatif` (e.g. `assets/pillars/anime.png`). Falls back to a
  CSS gradient built from that pillar's accent colors.

## Structure

- **Hub** — brand thesis, market line, the 4 pillar cards, and the full bilingual Brand
  Bible below.
- **Pilares (Pillars)** — grid of the 4 narrative pillars → pillar detail (accent hero,
  format notes, that pillar's cast and stories).
- **Elenco (Cast)** — roster filterable by pillar → character detail (portrait/medallion,
  alias, pillar tag, priority, role quote, full bilingual sections).
- **Historias (Stories)** — storyboard list filterable by pillar → storyboard viewer
  (logline, thesis, then every shot: slug/camera line, bilingual action, bilingual VO
  as a quote).
- **Producción (Production)** — visual language: palette, per-pillar accents, framing
  rules, recurring motifs (from `visual_language.md`).
- **Pipeline** — the six-stage production pipeline and the script→kit engine (from
  `production_pipeline.md`).

All colors are CSS variables in `:root` for a one-line re-skin; per-pillar/per-character
accent colors are data-driven (from `build_xsino.py`'s curation tables) rather than
hardcoded in CSS.

## Language toggle

**ES / AMBOS / EN** in the top bar, persisted in `localStorage` (`xsino_mode`), default
`AMBOS` (both languages side by side).
